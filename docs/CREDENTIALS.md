# agencios — Credentials structure

App-level secrets (API keys, OAuth client secrets, webhook secrets) live **only** in Rails encrypted
credentials. Per-workspace OAuth tokens are **not** here — they're `encrypts`-ed on the models
(`SocialAccount`, `Setting`). Edit with:

```bash
EDITOR=nano bin/rails credentials:edit          # decrypts with config/master.key
```

Every key below is read via `credential(:vendor, :key, env: "ENV_FALLBACK")` — so in **local dev** you
may instead set the `ENV_FALLBACK` (in `.env`) and skip credentials entirely. In **production**, use
credentials (or a secret manager). Anything missing degrades gracefully: AI falls back to a stub,
billing/calendar fall back to mock flows, a network simply can't publish until connected.

---

## Full `credentials.yml.enc` structure

```yaml
# Rails core
secret_key_base: <generated by `rails credentials:edit` on first run>

# Active Record Encryption — protects the encrypted token columns.
# (Currently the initializer also accepts AR_ENCRYPTION_* env vars with dev fallbacks;
#  in production set these here and delete the dev fallbacks.)
active_record_encryption:
  primary_key:          <32+ random chars>
  deterministic_key:    <32+ random chars>
  key_derivation_salt:  <32+ random chars>

# ─────────────────────────────────────────────────────────────
# Social networks — publishing + analytics (direct integration)
# ─────────────────────────────────────────────────────────────
meta:                          # Instagram + Facebook share ONE Meta app
  app_id:               "..."
  app_secret:           "..."
  webhook_verify_token: "..."  # the token you set on the Meta webhook subscription
  graph_version:        "v25.0" # optional — defaults to v25.0

tiktok:
  client_key:     "..."
  client_secret:  "..."
  webhook_secret: "..."

linkedin:
  client_id:     "..."
  client_secret: "..."

x:                             # X / Twitter (OAuth2 PKCE)
  client_id:     "..."
  client_secret: "..."

# ─────────────────────────────────────────────────────────────
# Creative generation + AI
# ─────────────────────────────────────────────────────────────
anthropic:                     # contextual ticket summaries, captions, retros
  api_key: "sk-ant-..."
  model:   "claude-sonnet-4-6"  # optional — model override

heygen:                        # UGC / avatar video (metered: video_generation)
  api_key:        "..."
  webhook_secret: "..."

image_gen:                     # carousel + image model (e.g. Replicate-style)
  api_key:        "..."
  base_url:       "..."        # optional — provider endpoint
  model_version:  "..."        # optional — model/version id

# ─────────────────────────────────────────────────────────────
# Google — ONE OAuth client for every Google surface: Sign in with
# Google, Calendar + Meet, and YouTube (Data API v3 + Analytics).
# There is no separate `youtube:` block — they share this client.
# ─────────────────────────────────────────────────────────────
google:
  client_id:     "...apps.googleusercontent.com"
  client_secret: "..."

# ─────────────────────────────────────────────────────────────
# SaaS billing — Stripe (charges the workspace)
# ─────────────────────────────────────────────────────────────
stripe:
  secret_key:     "sk_live_..."
  webhook_secret: "whsec_..."
  api_version:    "2025-03-31.basil"   # optional — Billing Meters need Basil+
  prices:                              # Stripe Price ids
    solo:                "price_..."   # licensed (1 seat)
    agencia:             "price_..."   # licensed, per-seat (5–20)
    enterprise:          "price_..."   # licensed, per-seat (20+)
    carousel_generation: "price_..."   # metered → carousel_generation meter
    video_generation:    "price_..."   # metered → video_generation meter

# ─────────────────────────────────────────────────────────────
# Client billing — Mercado Pago (the agency charges its clients)
# ─────────────────────────────────────────────────────────────
mercado_pago:
  access_token:   "APP_USR-..."  # platform token (Pix/boleto/card)
  client_id:      "..."          # for per-workspace OAuth (split payments) — optional now
  client_secret:  "..."          # optional now
  webhook_secret: "..."          # x-signature HMAC secret

# ─────────────────────────────────────────────────────────────
# Web Push (VAPID) — PWA browser notifications. Required in EVERY environment.
# ─────────────────────────────────────────────────────────────
vapid:
  public_key:  "..."
  private_key: "..."
```

> The committed template [`config/credentials.example.yml`](../config/credentials.example.yml) mirrors
> this structure key-for-key — use it as the canonical reference for every environment.

## ENV fallbacks (dev `.env`)

Each credential has a matching ENV var (used only if the credential is absent). For local dev you can
put these in `.env` instead of editing credentials:

| Credential | ENV fallback |
|---|---|
| `meta.app_id` / `app_secret` / `webhook_verify_token` | `META_APP_ID` / `META_APP_SECRET` / `META_WEBHOOK_VERIFY_TOKEN` |
| `tiktok.client_key` / `client_secret` / `webhook_secret` | `TIKTOK_CLIENT_KEY` / `TIKTOK_CLIENT_SECRET` / `TIKTOK_WEBHOOK_SECRET` |
| `linkedin.client_id` / `client_secret` | `LINKEDIN_CLIENT_ID` / `LINKEDIN_CLIENT_SECRET` |
| `x.client_id` / `client_secret` | `X_CLIENT_ID` / `X_CLIENT_SECRET` |
| `anthropic.api_key` / `model` | `ANTHROPIC_API_KEY` / `ANTHROPIC_MODEL` |
| `heygen.api_key` / `webhook_secret` | `HEYGEN_API_KEY` / `HEYGEN_WEBHOOK_SECRET` |
| `image_gen.api_key` / `base_url` / `model_version` | `IMAGE_GEN_API_KEY` / `IMAGE_GEN_BASE_URL` / `IMAGE_GEN_MODEL_VERSION` |
| `google.client_id` / `client_secret` (sign-in + Calendar + YouTube) | `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` |
| `stripe.secret_key` / `webhook_secret` | `STRIPE_SECRET_KEY` / `STRIPE_WEBHOOK_SECRET` |
| `mercado_pago.access_token` / `client_id` / `client_secret` / `webhook_secret` | `MERCADO_PAGO_ACCESS_TOKEN` / `MERCADO_PAGO_CLIENT_ID` / `MERCADO_PAGO_CLIENT_SECRET` / `MERCADO_PAGO_WEBHOOK_SECRET` |
| AR encryption | `AR_ENCRYPTION_PRIMARY_KEY` / `AR_ENCRYPTION_DETERMINISTIC_KEY` / `AR_ENCRYPTION_KEY_DERIVATION_SALT` |

> Stripe `prices.*` have **no** ENV fallback — set them in credentials (they're not single values).

## OAuth redirect URIs to register in each portal

`<APP_HOST>/auth/<slug>/callback` — where `slug` is `meta` (IG+FB), `tiktok`, `youtube`, `linkedin`, `x`.
Webhooks: `<APP_HOST>/webhooks/{stripe,mercadopago,heygen,meta}`.

## Generate fresh encryption keys

```bash
bin/rails db:encryption:init      # prints a ready-to-paste active_record_encryption block
```
